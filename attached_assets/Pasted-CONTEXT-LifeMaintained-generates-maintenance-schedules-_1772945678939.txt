CONTEXT: LifeMaintained generates maintenance schedules via the generate-maintenance-schedule Edge Function. The function receives vehicle_type which is actually the FUEL TYPE (gas/diesel/hybrid/ev) and is_awd (boolean). But it NEVER receives the VEHICLE CATEGORY (car/motorcycle/rv/boat/atv/utv/pwc/snowmobile). The maintenance_templates table has vehicle_type values of 'all', 'gas', 'diesel', 'ev', 'hybrid', 'awd_4wd' — all fuel-based, no category awareness.
This means a MOTORCYCLE gets ALL templates marked 'all' + 'gas', including absurd items like Tire Rotation (motorcycles don't rotate tires — they have one front and one rear, different sizes), Cabin Air Filter (no cabin), Wiper Blade Replacement (no wipers), Serpentine Belt (most use chain drive), Transmission Fluid Automatic (most are manual), and PCV Valve. Same problem for boats, ATVs, PWCs, snowmobiles — they all get car-centric templates.
This is a credibility-destroying bug. A motorcycle owner seeing "Tire Rotation" immediately loses trust in the entire app.

TASK: Add vehicle category awareness to the maintenance schedule system. This requires changes to the Edge Function, the add-vehicle call, and the maintenance_templates table.
