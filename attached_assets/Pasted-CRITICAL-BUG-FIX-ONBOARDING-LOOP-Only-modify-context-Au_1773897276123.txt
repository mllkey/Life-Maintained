CRITICAL BUG FIX — ONBOARDING LOOP. Only modify context/AuthContext.tsx. Do NOT modify any other files. Do NOT modify LogSheet, dashboard, or any visual code.
There are TWO bugs causing the onboarding loop:
BUG 1: AsyncStorage.getItem runs as fire-and-forget (.then) while fetchProfile controls isLoading. If fetchProfile resolves before AsyncStorage, the router sees onboardingCompleted=false and redirects to onboarding before the cache is read.
BUG 2: fetchProfile's success handler overwrites the cached onboardingCompleted value. If the DB returns onboarding_completed as null, it sets onboardingCompleted=false AND deletes the AsyncStorage cache.
FIX — Replace the entire INITIAL_SESSION/SIGNED_IN block with this logic:
if (session?.user) {
  userIdRef.current = session.user.id;
  setIsLoading(true);

  // STEP 1: Read AsyncStorage FIRST and WAIT for it before proceeding
  let cachedOnboarding = false;
  try {
    const cached = await AsyncStorage.getItem("@onboarding_completed");
    cachedOnboarding = cached === "true";
    if (cachedOnboarding && mountedRef.current) {
      setOnboardingCompleted(true);
    }
  } catch {}

  // STEP 2: Fetch profile from DB
  await fetchProfile(session.user.id);

  // STEP 3: Only NOW set isLoading to false
  if (mountedRef.current) setIsLoading(false);
}

SUMMARY OF WHAT THIS FIXES:

AsyncStorage is read BEFORE isLoading becomes false — no race condition
If DB says onboarding_completed=false but cache says true, the cache wins
The only way to reset onboarding is signOut (which clears the cache)
This makes the onboarding state PERMANENT once set, surviving network failures, DB issues, and hot reloads
