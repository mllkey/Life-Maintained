We need to add a "Wallet" feature that stores a vehicle's registration, insurance, and ID card info — like a digital glove box. This prompt creates the database table and the read-only display. The next prompt will add the add/edit forms and copy-from-vehicle flow.
IMPORTANT: This feature stores sensitive personal information (policy numbers, license numbers). All data must be stored in Supabase with RLS policies ensuring only the owning user can read/write. No sensitive data in AsyncStorage.

TASK: Create the wallet database table and add a Wallet tab to the vehicle detail screen that displays document cards.
STEP 1 — DATABASE: Create a table called "vehicle_wallet_documents" via Supabase migration.
Columns:

id: uuid, primary key, default gen_random_uuid()
user_id: uuid, not null, FK → auth.users(id) ON DELETE CASCADE
vehicle_id: uuid, not null, FK → vehicles(id) ON DELETE CASCADE
document_type: text, not null — one of: 'registration', 'insurance', 'id_card'
data: jsonb, not null
created_at: timestamptz, default now()
updated_at: timestamptz, default now()

RLS policies: SELECT, INSERT, UPDATE, DELETE all scoped to user_id = auth.uid().
Add index on (user_id, vehicle_id, document_type).
Add unique constraint on (vehicle_id, document_type) — one document per type per vehicle.
The JSONB "data" field stores different shapes per document_type:
Registration: { "state": "IL", "plate_number": "ABC1234", "registration_number": "REG123456", "expiration_date": "2027-03-15", "registered_owner": "John Doe" }
Insurance: { "provider": "State Farm", "policy_number": "POL-987654", "group_number": "", "coverage_type": "Full Coverage", "expiration_date": "2026-09-01", "agent_name": "Jane Smith", "agent_phone": "847-555-1234", "claims_phone": "800-555-0000" }
ID Card: { "full_name": "John Doe", "license_number": "D123-4567-8901", "state": "IL", "expiration_date": "2028-06-15", "date_of_birth": "1990-01-15", "class": "D" }
