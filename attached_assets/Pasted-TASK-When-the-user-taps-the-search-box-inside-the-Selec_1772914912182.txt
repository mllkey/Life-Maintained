TASK: When the user taps the search box inside the Select Model (or Select Make) bottom sheet, the keyboard appears and covers the entire list of results, making it impossible to see or tap any options. Fix this and audit the entire app for the same issue.
PART 1 — FIX THE MODEL/MAKE PICKER SHEETS
Find the Select Model and Select Make bottom sheet/modal components. The fix requires the list to scroll up and remain visible above the keyboard when it appears.
Implement KeyboardAvoidingView wrapping the sheet content:
javascriptimport { KeyboardAvoidingView, Platform } from 'react-native';

<KeyboardAvoidingView
  behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
  keyboardVerticalOffset={/* measure from bottom of sheet header to bottom of screen */}
>
  {/* search input */}
  {/* results list */}
</KeyboardAvoidingView>
Additionally, ensure the results list uses FlatList (not ScrollView) with these props set:
javascriptkeyboardShouldPersistTaps="handled"
keyboardDismissMode="on-drag"
contentContainerStyle={{ paddingBottom: 20 }}
keyboardShouldPersistTaps="handled" is critical — without it, tapping a result first dismisses the keyboard instead of registering the tap, requiring the user to tap twice.
The search input itself must NOT use autoFocus — keyboard must only open when the user explicitly taps the search box, never automatically. If autoFocus is present on these inputs, remove it.
