CONTEXT: LifeMaintained is a React Native / Expo app with push notifications already partially built. The notification scheduler (lib/notificationScheduler.ts) and notification settings screen (app/notifications-settings.tsx) exist and are functional. However, there are two bugs preventing notifications from working correctly:

BUG 1: The notification scheduler queries the WRONG TABLE for vehicle maintenance tasks. It currently queries "vehicle_maintenance_tasks" (line ~94 in notificationScheduler.ts), but the actual data lives in "user_vehicle_maintenance_tasks". It also selects column "task" but the correct column name in user_vehicle_maintenance_tasks is "name".

BUG 2: app.json is missing the expo-notifications plugin configuration, which means iOS push notification permissions will not work on a real device.

IMPORTANT: These are small, targeted fixes. Do NOT rewrite the notification scheduler or settings screen. Only fix the specific issues listed below.

TASK: Fix two bugs so push notifications work correctly.
