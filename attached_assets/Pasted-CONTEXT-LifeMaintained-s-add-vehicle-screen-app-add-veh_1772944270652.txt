CONTEXT: LifeMaintained's add-vehicle screen (app/add-vehicle.tsx) loads models from the NHTSA API using GetModelsForMakeYear. When vehicleType is "car" (which covers Cars, Trucks, and SUVs), it queries two NHTSA vehicleType categories: "Passenger Car" and "Truck". However, some common vehicles are missing from the model picker because NHTSA categorizes them under different vehicleType values. For example:

A 2004 Dodge Ram may not show "Ram 1500", "Ram 2500", "Ram 3500" because NHTSA may categorize them under "Multipurpose Passenger Vehicle (MPV)" instead of "Truck"
SUVs like the Ford Explorer, Chevy Tahoe, Jeep Grand Cherokee may be under "MPV" not "Truck" or "Passenger Car"
Some crossovers and minivans may only appear under "MPV"
Vans (Transit, Sprinter, Express) may be under "Bus" or "Incomplete Vehicle"

This means a significant number of the most popular US vehicles are invisible in the model picker, forcing users to type a custom model name — which most users won't think to do.

FIX 3 — Add a fallback "all types" query:
After the type-specific queries complete, if the resulting model list has FEWER than 3 models, run a fallback query with NO vehicleType filter to catch anything NHTSA categorized unexpectedly:
if (names.length < 3) {
  const allResp = await fetch(`${nhtsaBase}?format=json`);
  const allJson = await allResp.json();
  names = [...new Set([...names, ...extractNames(allJson)])];
}
This ensures that even if NHTSA recategorizes vehicles, users always see a complete list. The threshold of 3 ensures the fallback only fires when the primary queries came up nearly empty.
